# DSL Manifest Schema

Source of truth: `packages/dsl/src/schema.ts` (Zod). TypeScript types are inferred from the
Zod schemas — never defined separately. Reference: `docs/DSL/dsl-schema-reference.md`
v3.2 for worked examples and YAML snippets.

---

## Top-level fields

### Required

| Field | Type | Validation | Description |
|---|---|---|---|
| `name` | `string` | `min(1)` | Unique MFE identifier. Used as the registry key and as the generated class/module name. |
| `version` | `string` | semver pattern `\d+\.\d+\.\d+` | Manifest version. Bump on any breaking change to capability inputs/outputs. |
| `type` | `MFEType` | enum | Architectural role. See [MFEType enum](#mfetype-enum) below. |
| `language` | `Language` | enum | Implementation language. See [Language enum](#language-enum) below. |
| `capabilities` | `CapabilityEntry[]` | non-empty array | One or more named capability configs. See [Capabilities](#capabilities). |

### Optional — identity

| Field | Type | Default | Description |
|---|---|---|---|
| `description` | `string` | — | Human-readable summary shown in `describe` output. |
| `owner` | `string` | — | Team or individual responsible. Used for impact analysis (ADR-008). |
| `tags` | `string[]` | — | Arbitrary labels (e.g. `[pii, gdpr-relevant]`). Registry search, ADR-008. |
| `category` | `string` | — | Domain grouping for the marketplace / discovery UI. |
| `framework` | `string (min 1)` | `'react'`* | UI framework. **Open string** — unknown values emit a warning, not an error (ADR-036). Known built-ins: `react`, `angular`. |
| `bundler` | `string (min 1)` | `'rspack'`* | Build tool. **Open string** — same policy as `framework` (ADR-036). Known built-ins: `rspack`, `webpack`. |

*When omitted, codegen defaults to `react` + `rspack` for backward compatibility.

### Optional — endpoints

| Field | Type | Validation | Description |
|---|---|---|---|
| `endpoint` | `string` | URL | Base URL where the daemon reaches capability endpoints (e.g. `http://my-mfe:3000`). Required at runtime; generated by `remote:init`. |
| `remoteEntry` | `string` | URL | Module Federation remote entry URL (e.g. `http://my-mfe:3000/remoteEntry.js`). |
| `discovery` | `string` | URL | URL to the MFE's discovery/health endpoint. |

### Optional — sections

| Field | Type | Description |
|---|---|---|
| `dependencies` | `Dependencies` | Runtime, design-system, and MFE-to-MFE dependency declarations. |
| `data` | `DataConfig` | GraphQL Mesh BFF configuration. See [Data section](#data-section). |
| `providesSlots` | `ProvidedSlot[]` | Named layout regions this MFE contributes to a host. See [Provided slots](#provided-slots). |
| `performance` | `PerformanceConfig` | Caching, observability, rate limiting. See [Performance section](#performance-section). |
| `transforms` | `string[]` | YAML strings for custom resolver composition. |
| `authorization` | `unknown` | Deferred — ADR-007. Reserved field; no validation applied yet. |

## Provided slots

`providesSlots` declares the local slot ids an MFE may register through the
host-provided `provideSlot` callback (ADR-067).

```yaml
providesSlots:
  - id: main
    description: Primary content region
  - id: card.{sku}
    description: Stable repeated region keyed by product SKU
```

### ProvidedSlot

| Field | Type | Required | Validation |
|---|---|---|---|
| `id` | `string` | yes | Unique within the manifest. Dot-separated segments are assigned-name literals containing a letter or `{param}` placeholders. `/`, empty segments, and purely numeric segments are rejected. |
| `description` | `string` | no | Human-readable purpose for discovery and rule-authoring tools. |

The MFE registers the local id (`main`). The host builds the public address
from `RenderedExperience.mfe`, for example `catalog-layout/main` (ADR-068).
Codegen emits a manifest-bound slot helper for both supported frameworks:

- React: `src/slots.tsx` with `DeclaredSlot`.
- Angular: `src/slots.ts` with standalone `DeclaredSlotDirective`
  (`[smtDeclaredSlot]`).

The grammar is defined once in `@seans-mfe/contracts` and consumed by both
the DSL validator and runtime matcher (ADR-069). See
[The Slot Contract](../slot-contract.md).

## MFEType enum

```
'tool' | 'agent' | 'feature' | 'service' | 'remote' | 'shell' | 'bff'
```

| Value | Role |
|---|---|
| `tool` | CLI utility or automation MFE |
| `agent` | AI/reasoning agent |
| `feature` | Domain feature MFE (most common) |
| `service` | Background service or BFF-only MFE |
| `remote` | Module Federation remote (delivers UI to a shell) |
| `shell` | Host shell that mounts remotes |
| `bff` | Backend-for-frontend only (no UI rendering) |

## Language enum

```
'javascript' | 'typescript' | 'python' | 'go' | 'rust' | 'java'
```

Determines codegen template variant and import paths. The platform's own runtime is
TypeScript; other languages require community adaptors.

## Capabilities

`capabilities` is an array of single-key objects mapping a capability name to a
`CapabilityConfig`. This allows ordered, named capabilities without duplicate-key
ambiguity in YAML.

```yaml
capabilities:
  - PlayGame:
      type: domain
      description: Launch and manage the game session
      inputs:
        - name: gameId
          type: id!
      lifecycle:
        before:
          - validate: { handler: validateGameId }
```

### CapabilityConfig

| Field | Type | Required | Description |
|---|---|---|---|
| `type` | `'platform' \| 'domain'` | yes | `platform` = built-in lifecycle capability; `domain` = MFE-specific business capability. |
| `description` | `string` | no | Human-readable purpose. |
| `handler` | `string` | no | Default handler function name for the capability body. |
| `inputs` | `DSLInput[]` | no | Capability input parameters. See [DSLInput](#dslinput). |
| `outputs` | `DSLOutput[]` | no | Declared output schema. See [DSLOutput](#dsloutput). |
| `lifecycle` | `Lifecycle` | no | Before/main/after/error hook phases. See [Lifecycle](#lifecycle). |
| `authorization` | `string` | no | Deferred (ADR-007). Reserved. |

### Platform capabilities

Every MFE inherits these 9 platform capabilities from `BaseMFE`. They do not need to
be declared in the manifest unless you are overriding lifecycle hooks:

`load` · `render` · `refresh` · `authorizeAccess` · `health` · `describe` · `schema` · `query` · `emit`

### DSLInput

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | `string` | yes | Parameter name. |
| `type` | `string` | yes | DSL type string. See [DSL type system](dsl-types.md). |
| `description` | `string` | no | Parameter documentation. |
| `default` | `unknown` | no | Default value when input is omitted. |
| `values` | `string[]` | no | Allowed values for enum-style inputs. |
| `formats` | `string[]` | no | Allowed MIME types for `file` inputs. |

### DSLOutput

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | `string` | yes | Output field name. |
| `type` | `string` | yes | DSL type string. See [DSL type system](dsl-types.md). |
| `description` | `string` | no | Output field documentation. |

## Lifecycle

Lifecycle hooks execute around a capability invocation. Each phase is an ordered array
of hook entries.

| Phase | When it runs | Failure behaviour |
|---|---|---|
| `before` | Before the capability handler | If a `mandatory` hook fails, the capability is aborted |
| `main` | The capability handler itself | N/A — this is the core invocation |
| `after` | After successful handler completion | Non-mandatory failures are logged but do not roll back |
| `error` | On any unhandled error in `before` or `main` | Always attempted; errors here are surfaced as telemetry |

### LifecycleHook

| Field | Type | Required | Description |
|---|---|---|---|
| `handler` | `string \| string[]` | yes | Handler name(s). Must NOT be a platform wrapper method (`doLoad`, `doRender`, etc.). |
| `source` | `string` | no | Module specifier for the handler — relative path (`./handlers/foo.ts`), package (`@org/pkg`), or package + named export (`@org/pkg#exportName`). See ADR-040. |
| `description` | `string` | no | Purpose of this hook. |
| `mandatory` | `boolean` | no | If `true`, the capability aborts when this hook fails. Default: `false`. |
| `contained` | `boolean` | no | If `true`, hook errors are isolated and do not propagate. Default: `false`. |

## Data section

The `data` section configures the GraphQL Mesh BFF layer (ADR-012). Omit it entirely
for non-BFF MFEs.

### DataConfig

| Field | Type | Required | Description |
|---|---|---|---|
| `sources` | `DataSource[]` | yes | Upstream API specifications to federate. |
| `transforms` | `DataTransform[]` | no | Schema shaping rules applied to sources. |
| `plugins` | `DataPlugin[]` | no | Production plugins (caching, observability, etc.). |
| `serve` | `DataServe` | no | BFF server endpoint configuration. |
| `mockSwitch` | `MockSwitch` | no | Demo-mode live/mock switching (ADR-052). |
| `generatedFrom` | `DataLineage[]` | no | Upstream API provenance for impact analysis (ADR-011). |

### DataSource

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | `string` | yes | Logical source name used in transforms. |
| `handler.openapi.source` | `string` | yes | Path or URL to the OpenAPI spec. |
| `handler.openapi.operationHeaders` | `Record<string, string>` | no | Headers forwarded to every operation (e.g. `Authorization`). |
| `transforms` | `Record<string, unknown>[]` | no | Source-level transforms applied before merging. |

### DataServe

| Field | Type | Default | Description |
|---|---|---|---|
| `endpoint` | `string` | — | GraphQL endpoint path, e.g. `/graphql`. |
| `playground` | `boolean` | — | Enable GraphQL Playground in development. |

### MockSwitch

| Field | Type | Default | Description |
|---|---|---|---|
| `enabled` | `boolean` | `false` | When `true`, per-request live/mock switching is enabled (ADR-052). |

### DataLineage

| Field | Type | Description |
|---|---|---|
| `openapi` | `string` | Path to the OpenAPI spec this data section was generated from. |
| `service` | `string` | Logical service name. |
| `version` | `string` | API version at time of generation. |

### Known transforms and plugins

Validation warns if a transform name is used in `plugins` (or vice versa).

**Transforms:** `filterSchema` · `rateLimit` · `rename` · `prefix` · `encapsulate` · `federation` · `namingConvention` · `cache` · `snapshot` · `mock` · `resolversComposition` · `type-merging`

**Plugins:** `prometheus` · `useMaskedErrors` · `useResponseCache` · `usePersistedOperations` · `newrelic` · `datadog` · `statsd` · `mock` · `snapshot`

## Performance section

### PerformanceConfig

| Field | Type | Description |
|---|---|---|
| `caching` | `CachingConfig` | Response cache settings. |
| `observability` | `ObservabilityConfig` | Prometheus and OpenTelemetry. |
| `rateLimit` | `RateLimitConfig` | Per-operation rate limits. |

### CachingConfig

| Field | Type | Default | Description |
|---|---|---|---|
| `enabled` | `boolean` | `true` | Toggle response caching. |
| `ttl` | `number` | `300000` | Cache TTL in milliseconds (default 5 min). |
| `strategies` | `{type, field, ttl}[]` | — | Per-field TTL overrides. |

### PrometheusConfig

| Field | Type | Default | Description |
|---|---|---|---|
| `enabled` | `boolean` | `true` | Expose Prometheus metrics. |
| `port` | `number` | `9090` | Metrics server port. |
| `endpoint` | `string` | `'/metrics'` | Metrics endpoint path. |

### OpenTelemetryConfig

| Field | Type | Default | Description |
|---|---|---|---|
| `enabled` | `boolean` | `false` | Enable OpenTelemetry tracing. |
| `serviceName` | `string` | — | Service name in traces. |
| `sampling.probability` | `number [0–1]` | `0.1` | Trace sampling rate. |
| `exporters` | `{type, endpoint}[]` | — | Exporter targets (e.g. Jaeger). |

### RateLimitConfig

| Field | Type | Default | Description |
|---|---|---|---|
| `enabled` | `boolean` | `false` | Enable rate limiting. |
| `config` | `RateLimitRule[]` | — | Per-operation rules. |

**RateLimitRule fields:** `type` (Query/Mutation), `field` (field name or `*`), `max` (requests), `ttl` (window ms), `identifyContext` (optional context field for per-user limits).

## Validation rules summary

| Rule | Where enforced |
|---|---|
| `name` non-empty | Zod `min(1)` |
| `version` is semver | Zod regex |
| `type` is known enum | Zod `z.enum(...)` — hard error |
| `language` is known enum | Zod `z.enum(...)` — hard error |
| `framework` / `bundler` unknown | Warn on stderr, not error (ADR-036) |
| Handler is not a platform wrapper | Zod `refine` on `LifecycleHookSchema` |
| Plugin name in transforms (and vice versa) | Zod `superRefine` |
| `endpoint` / `remoteEntry` / `discovery` are URLs | Zod `z.string().url()` |
| `sampling.probability` in [0, 1] | Zod `z.number().min(0).max(1)` |
