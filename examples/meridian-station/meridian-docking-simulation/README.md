# meridian-docking-simulation - GraphQL BFF

Generated by [seans-mfe-tool](https://github.com/falese/seans-mfe-tool) following ADR-012.

## Overview

This is a GraphQL BFF (Backend for Frontend) that provides a unified GraphQL interface to your REST APIs using [GraphQL Mesh](https://the-guild.dev/graphql/mesh).

## Quick Start

```bash
# Install dependencies
npm install

# Development mode (hot reload)
npm run dev

# Build for production
npm run build

# Start production server
npm start
```

> **Heads-up: `@seans-mfe-tool/runtime` is not published yet** (ADR-064).
> A plain `npm install` fails resolving it. Until the runtime ships to npm,
> stage it from a local checkout of
> [seans-mfe-tool](https://github.com/falese/seans-mfe-tool):
>
> ```bash
> # 1. Build the runtime once in your checkout
> (cd /path/to/seans-mfe-tool && npm ci && npm run build)
> # 2. Point the dep at it, install, then replace the symlink with a real copy
> npm pkg set 'devDependencies[@seans-mfe-tool/runtime]=file:/path/to/seans-mfe-tool/dist/runtime'
> npm install --legacy-peer-deps
> rm -rf node_modules/@seans-mfe-tool/runtime
> mkdir -p node_modules/@seans-mfe-tool
> cp -r /path/to/seans-mfe-tool/dist/runtime node_modules/@seans-mfe-tool/runtime
> ```
>
> The final copy must be a **real directory**, not a `file:` symlink вҖ” a
> symlink makes module resolution escape the project, so Angular/rspack builds
> stop resolving framework imports. The generated Dockerfile already stages
> the runtime this way for you.

## Configuration

The GraphQL Mesh configuration is embedded in `mfe-manifest.yaml` under the `data:` section. This is the single source of truth (ADR-012).

### Adding a New API Source

Edit `mfe-manifest.yaml`:

```yaml
data:
  sources:
    - name: NewAPI
      handler:
        openapi:
          source: ./specs/new-api.yaml
          operationHeaders:
            Authorization: 'Bearer {context.jwt}'
```

Then rebuild:

```bash
npm run mesh:build
```
